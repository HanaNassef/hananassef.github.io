AuraViT-FL (Graduation Project)

Designed a privacy-preserving medical imaging framework using Federated Learning. The system allows multiple hospitals to collaboratively train a segmentation model without sharing patient data, achieving state-of-the-art accuracy.

Deep Learning Federated Learning Transformers Medical Imaging
View Code View Paper

Project Overview

Grade: A+ (Top of Class). This project addresses the challenge of data privacy in medical AI. We developed "AuraViT-FL," a resource-efficient hybrid transformer architecture optimized for Federated Learning settings.

Technical Contributions

  • Hybrid Architecture: Combined the local feature extraction of CNNs with the global context understanding of Vision Transformers (ViTs).
  • Federated Strategy: Implemented the FedAvg algorithm to aggregate model updates from simulated hospital clients, ensuring raw data never left the local device.
  • Optimization: Reduced computational cost significantly compared to standard ViTs, making it suitable for edge deployment in hospitals with limited hardware.
  • Results: Achieved superior Dice scores on lung tumor segmentation benchmarks compared to standalone UNet and Swin-UNet models.

Book Recommendation System

Python Scikit-learn Flask
View on GitHub Watch Demo

Project Overview

This project is a complete book recommendation system, from data cleaning to a full-stack web application. It serves as a strong demonstration of a machine learning pipeline, API development, and data visualization. The system leverages content-based filtering, recommending books with similar genres and descriptions.

Key Technologies & Concepts

  • Python: The foundational language for the entire project, used for both the machine learning pipeline and the Flask API.
  • Scikit-learn: Utilized for the core recommendation logic, including TF-IDF vectorization and cosine similarity, to measure the semantic similarity between books.
  • Flask: A lightweight web framework used to build a robust back-end API that communicates with the front-end to serve real-time recommendations.
  • Dash: Used to create the **dynamic and interactive dashboard** for the exploratory data analysis (EDA) notebook, showcasing data visualizations in a clean, web-based format.
  • Pandas: Employed for all data manipulation, cleaning, and preprocessing tasks, from loading the dataset to preparing it for the recommendation model.
  • FuzzyWuzzy: An essential library for implementing **fuzzy matching**, which enables the application to handle user typos in book titles and author names.

Project Overview

This project applies ensemble learning techniques to the UNSW-NB15 dataset for network intrusion detection. A key challenge was the severe class imbalance, which was addressed by creating balanced samples for training and testing.

Workflow

  • Created balanced training and testing sets using stratified sampling.

C++ OpenMP AutoParallelizer

Developed a C++ tool to automatically parallelize 'for' loops by injecting OpenMP directives. Evaluated performance using Intel® VTune™ Profiler.

C++ OpenMP Intel VTune
View on GitHub

About the Project

Manual parallelization is complex and error-prone. This C++ source-to-source compiler tool automatically parallelizes 'for' loops using OpenMP to leverage the power of modern multi-core processors, simplifying parallel programming.

How It Works

  • The user provides a C++ source file.
  • The tool scans for variables declared outside of loops to identify shared variables.
  • It detects 'for' loops and automatically inserts `#pragma omp parallel for`.
  • It adds `#pragma omp atomic` for shared variable updates inside loops to ensure thread safety.
